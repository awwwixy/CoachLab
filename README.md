# CoachLab — онлайн-платформа тренера

Сайт зроблено на простому стеку без складних фреймворків:
**HTML + CSS + JavaScript** на стороні браузера та **Node.js + Express** на сервері.
Уся інформація зберігається у файлі **db.json** (це наша «база даних»).

---

## 🚀 Як запустити сайт

Потрібен встановлений [Node.js](https://nodejs.org) (версія 16 або новіша).

> 💡 Бібліотека Express уже включена в папку `node_modules`, тому крок `npm install`
> можна пропустити — сайт запуститься одразу. Якщо папки `node_modules` раптом немає
> (наприклад, ти переносила проєкт без неї), виконай `npm install` один раз.

1. Відкрий папку проєкту в терміналі.
2. (Необов'язково) Встанови залежності:

   ```bash
   npm install
   ```

3. Запусти сервер:

   ```bash
   npm start
   ```

   (це те саме, що `node server.js`)

4. Відкрий у браузері адресу:

   ```
   http://localhost:3000
   ```

Щоб зупинити сервер — натисни `Ctrl + C` у терміналі.

## 🧠 Логіка роботи сайту

1. **Реєстрація / вхід.** Браузер надсилає дані на сервер. Сервер перевіряє їх і,
   якщо все гаразд, повертає дані користувача. Браузер запам'ятовує, хто увійшов,
   у `localStorage` (це сховище браузера, дані якого не зникають після оновлення).

2. **Захист кабінету.** Якщо людина не увійшла, сторінка `dashboard.html` одразу
   перекидає її на сторінку входу.

3. **Уроки.** Перший урок безкоштовний. Решта заблоковані, доки курс не оплачено.

4. **Оплата (симуляція).** Користувач вводить дані картки у модальному вікні та
   натискає «Оплатити». Сервер просто позначає курс як куплений (`paid: true`).
   Після цього всі уроки розблоковуються.

5. **Відеоуроки.** Кнопка «Дивитися урок» відкриває модальне вікно з відеоплеєром.
   Посилання на відео беруться з файлу `db.json` (поле `video`).

6. **Прогрес.** На кожному доступному уроці є галочка «Виконано». Прогрес рахується
   автоматично: кругова діаграма, кількість виконаних уроків і відсоток.

7. **Завдання.** Користувач пише мету курсу та план тренувань на 7 днів і зберігає
   відповіді. Третє завдання — відмічати виконані уроки.

8. **Сповіщення.** З'являються при відкритті доступу до курсу та при виконанні уроку.
   Показуються після натискання на іконку дзвіночка.

Усі зміни (оплата, прогрес, відповіді, сповіщення) **зберігаються у файлі `db.json`**,
тому після оновлення сторінки нічого не зникає.

## 💳 Що таке «симуляція оплати»

Це **не справжня** оплата. Гроші нікуди не списуються, платіжна система (Stripe,
LiqPay тощо) не підключена. Коли користувач вводить дані картки й натискає «Оплатити»,
сайт лише перевіряє формат полів, показує повідомлення **«Оплата успішна»** і
позначає курс як куплений у базі `db.json`.

Так зроблено навмисно: для навчального проєкту не можна (і не треба) працювати з
реальними грошима. Якщо в майбутньому захочеш підключити справжню оплату — достатньо
буде замінити маршрут `/api/pay` на звернення до реального платіжного сервісу.

## 🎬 Як замінити відео на свої

Відкрий файл `db.json`, знайди потрібний урок і зміни поле `video` на своє посилання
(наприклад, пряме посилання на `.mp4`):

```json
{
  "id": 1,
  "title": "Урок 1. Вступ",
  "video": "ТВОЄ_ПОСИЛАННЯ_НА_ВІДЕО.mp4"
}
```

Прев'ю-картинку уроку можна замінити, поклавши свій файл у папку `public/images/`
і вказавши його у полі `image`.

## 🎓 Що показати викладачу на захисті

1. **Реєстрація та вхід** — створи акаунт або увійди демо-акаунтом; покажи, що ім'я
   з'являється у кабінеті, а без входу в кабінет не пускає.
2. **Оплата** — натисни «Оплатити курс», заповни поля картки, покажи повідомлення
   «Оплата успішна» і те, що уроки розблокувалися.
3. **Збереження даних** — онови сторінку (F5) і покажи, що доступ до курсу та прогрес
   нікуди не зникли (бо все зберігається у `db.json`).
4. **Відеоурок** — відкрий урок у модальному вікні з плеєром.
5. **Прогрес** — постав кілька галочок «Виконано» і покажи, як змінюється кругова
   діаграма та відсоток.
6. **Завдання** — заповни мету курсу та план тренувань, натисни «Зберегти відповідь».
7. **Сповіщення** — натисни на дзвіночок і покажи список сповіщень.
8. **Іконки кабінету** — покажи меню профілю та кнопку «Вийти».

## ⚠️ Важливо (чесне зауваження для захисту)

Це навчальний проєкт, тому деякі речі спрощені:

- паролі зберігаються у `db.json` як звичайний текст (у реальному проєкті їх шифрують);
- замість справжньої бази даних (наприклад, MySQL) використовується файл `db.json`;
- оплата — це симуляція без справжньої платіжної системи.

Ці спрощення зроблені свідомо, щоб код залишався простим і зрозумілим.
